# 网上通讯录系统 - 部署文档

## 1. 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 17+ | Java运行环境 |
| Maven | 3.8+ | 后端构建工具 |
| MySQL | 8.0+ | 数据库 |
| Node.js | 18+ | 前端运行环境 |
| npm | 9+ | 前端包管理 |

## 2. 数据库部署

### 2.1 创建数据库
登录MySQL，执行初始化脚本：

```bash
mysql -u root -p < sql/init.sql
```

或在MySQL客户端中执行：

```sql
source /path/to/sql/init.sql;
```

### 2.2 验证数据库
```sql
USE contact_system;
SHOW TABLES;
-- 应显示: contact, major, sys_user

SELECT * FROM major;
-- 应显示6条专业数据
```

## 3. 后端部署

### 3.1 修改数据库配置
编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/contact_system?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false
    username: root        # 修改为你的MySQL用户名
    password: root        # 修改为你的MySQL密码
```

### 3.2 构建后端
```bash
cd backend
mvn clean package -DskipTests
```

构建成功后，在 `target/` 目录下生成 `contact-system-1.0.0.jar`

### 3.3 启动后端
```bash
# 方式一：Maven直接运行（开发环境）
cd backend
mvn spring-boot:run

# 方式二：运行JAR包（生产环境）
java -jar target/contact-system-1.0.0.jar
```

后端启动成功后，访问 http://localhost:8080

**注意：** 首次启动时，系统会自动：
- 创建管理员账号（admin / admin123）
- 初始化专业数据

## 4. 前端部署

### 4.1 安装依赖
```bash
cd frontend
npm install
```

### 4.2 配置代理（开发环境）
`vite.config.js` 已配置API代理：

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

### 4.3 启动开发服务器
```bash
npm run dev
```

访问 http://localhost:5173

### 4.4 构建生产版本
```bash
npm run build
```

构建产物在 `dist/` 目录，可部署到Nginx等Web服务器。

## 5. Nginx部署（生产环境）

### 5.1 Nginx配置示例
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 5.2 启动Nginx
```bash
nginx -t          # 测试配置
nginx             # 启动
nginx -s reload   # 重载配置
```

## 6. 云服务器部署

### 6.1 服务器准备
```bash
# 安装JDK
apt install openjdk-17-jdk

# 安装MySQL
apt install mysql-server

# 安装Nginx
apt install nginx

# 安装Node.js (用于构建前端)
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install nodejs
```

### 6.2 上传项目
```bash
# 使用scp上传
scp -r contact-system/ root@your-server:/opt/
```

### 6.3 启动服务
```bash
# 启动MySQL
systemctl start mysql

# 导入数据库
mysql -u root -p < /opt/contact-system/sql/init.sql

# 启动后端（后台运行）
cd /opt/contact-system/backend
nohup java -jar target/contact-system-1.0.0.jar > app.log 2>&1 &

# 启动Nginx
systemctl start nginx
```

### 6.4 防火墙配置
```bash
# 开放80端口
ufw allow 80/tcp
ufw allow 443/tcp
```

## 8. 常见问题

### Q1: 后端启动失败，提示数据库连接错误
- 检查MySQL是否启动
- 检查数据库用户名密码是否正确
- 检查数据库contact_system是否已创建

### Q2: 前端页面空白
- 检查后端是否启动成功
- 检查浏览器控制台是否有报错
- 检查Vite代理配置是否正确

### Q3: 登录后提示"未登录或登录已过期"
- 检查JWT密钥配置
- 清除浏览器localStorage重新登录

### Q4: 管理员无法登录
- 首次启动时系统会自动创建管理员
- 如果SQL中已插入admin记录，启动类会自动更新密码
- 可以删除sys_user表中admin记录，重启应用重新创建

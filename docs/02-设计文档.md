# 网上通讯录系统 - 设计文档

## 1. 系统架构设计

### 1.1 总体架构
本系统采用前后端分离的B/S架构：

```
┌─────────────────────────────────────────────────────────────────┐
│                         浏览器（客户端）                          │
│                    Vue 3 + Element Plus + Vite                   │
└─────────────────────────────┬───────────────────────────────────┘
                              │ HTTP / RESTful API
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Nginx / 反向代理                            │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                   Spring Boot 应用服务器                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────────┐   │
│  │Controller│→ │ Service  │→ │  Mapper  │→ │   MySQL DB   │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────────┘   │
│       ↑                                                         │
│  ┌──────────┐                                                   │
│  │JWT拦截器 │                                                   │
│  └──────────┘                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 | 渐进式JavaScript框架，Composition API |
| UI组件库 | Element Plus | 基于Vue 3的组件库 |
| 构建工具 | Vite | 下一代前端构建工具 |
| HTTP客户端 | Axios | 基于Promise的HTTP库 |
| 状态管理 | Pinia | Vue 3官方推荐状态管理 |
| 后端框架 | Spring Boot 3.2 | Java企业级开发框架 |
| ORM框架 | MyBatis-Plus 3.5 | 增强版MyBatis |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| 认证方案 | JWT | 无状态Token认证 |
| 密码加密 | BCrypt | 单向哈希加密 |

## 2. 接口设计

### 2.1 认证接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 注册 | POST | /api/auth/register | 学生注册 |
| 登录 | POST | /api/auth/login | 用户登录 |
| 获取用户信息 | GET | /api/auth/info | 获取当前登录用户信息 |
| 修改密码 | PUT | /api/auth/password | 修改登录密码 |

### 2.2 通讯录接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取我的通讯录 | GET | /api/contact/my | 获取当前用户通讯录信息 |
| 保存通讯录 | POST | /api/contact/my | 保存/更新通讯录信息 |
| 查询通讯录 | POST | /api/contact/list | 分页查询通讯录列表 |

### 2.3 用户管理接口（管理员）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 用户列表 | GET | /api/admin/user/list | 分页查询用户列表 |
| 审核通过 | PUT | /api/admin/user/approve/{id} | 通过用户审核 |
| 禁用账号 | PUT | /api/admin/user/disable/{id} | 禁用用户账号 |
| 启用账号 | PUT | /api/admin/user/enable/{id} | 启用用户账号 |
| 删除用户 | DELETE | /api/admin/user/delete/{id} | 删除未审核用户 |

### 2.4 专业管理接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 专业列表 | GET | /api/major/list | 查询所有专业 |
| 新增专业 | POST | /api/major | 新增专业信息 |
| 修改专业 | PUT | /api/major | 修改专业信息 |
| 删除专业 | DELETE | /api/major/{id} | 删除专业信息 |

## 3. 安全设计

### 3.1 认证流程
```
用户登录 → 验证用户名密码 → 生成JWT Token → 返回Token
    ↓
前端存储Token → 每次请求携带 Authorization: Bearer <token>
    ↓
后端JwtInterceptor拦截 → 验证Token有效性 → 解析用户信息 → 放行/拒绝
```

### 3.2 权限控制
- 登录/注册接口：无需认证
- 学生功能接口：需要有效Token
- 管理员功能接口：需要有效Token + ADMIN角色

### 3.3 密码安全
- 使用BCrypt算法加密，自动加盐
- 密码不在网络中传输明文
- 前端使用password类型输入框

# 网上通讯录系统 - 需求分析文档

## 1. 项目概述

### 1.1 项目背景
本项目为南昌大学《Web程序设计(Java EE)》课程大作业，旨在为学院历届各专业学生建立一个网上通讯录系统，方便校友之间的联系与交流。

### 1.2 项目目标
- 实现学生注册、管理员审核的账号管理机制
- 提供学生通讯录信息的录入、修改、查询功能
- 实现管理员对用户账号和专业信息的管理功能
- 记录用户登录信息（登录次数、最近登录时间）

## 2. 功能需求

### 2.1 用户角色
系统包含两种角色：
- **学生（STUDENT）**：注册账号，完善通讯录信息，查询其他同学通讯录
- **管理员（ADMIN）**：审核学生注册，管理用户账号，维护专业信息

### 2.2 功能模块

#### 2.2.1 认证模块
| 功能 | 描述 |
|------|------|
| 用户注册 | 学生填写用户名、密码、真实姓名进行注册，注册后状态为"待审核" |
| 用户登录 | 用户输入用户名密码登录，系统验证账号状态（待审核/已禁用/正常） |
| 修改密码 | 登录用户可修改自己的登录密码 |
| 退出登录 | 清除登录状态，返回登录页 |

#### 2.2.2 学生功能模块
| 功能 | 描述 |
|------|------|
| 完善通讯录 | 填写/修改个人通讯录信息（专业、班级、入校年份、毕业年份、就业单位、城市、联系方式、邮箱） |
| 查询通讯录 | 按姓名、专业、班级、入校年份、城市、就业单位等条件查询其他同学的通讯录信息 |
| 查看首页 | 查看系统统计信息和个人信息 |

#### 2.2.3 管理员功能模块
| 功能 | 描述 |
|------|------|
| 用户审核 | 查看待审核用户列表，通过或拒绝学生注册申请 |
| 用户管理 | 禁用已通过审核的账号，启用已禁用的账号 |
| 用户删除 | 删除未通过审核的账户（已通过审核的账户不可删除） |
| 专业管理 | 增删改查专业信息（专业名称、所属学院） |

### 2.3 通讯录信息字段
| 字段 | 说明 |
|------|------|
| 专业 | 学生所读专业（关联专业表） |
| 班级 | 所在班级 |
| 入校年份 | 入学年份 |
| 毕业年份 | 毕业年份 |
| 就业单位 | 当前工作单位 |
| 所在城市 | 当前所在城市 |
| 联系方式 | 手机号码 |
| 电子邮箱 | 邮箱地址 |

### 2.4 系统记录信息
| 字段 | 说明 |
|------|------|
| 登录次数 | 记录用户累计登录次数 |
| 最近登录时间 | 记录用户最近一次登录的时间 |

## 3. 非功能需求

### 3.1 安全性
- 用户密码使用 BCrypt 加密存储
- 使用 JWT Token 进行身份认证
- 接口权限控制，防止越权访问

### 3.2 可用性
- 界面简洁美观，操作便捷
- 支持主流浏览器（Chrome、Firefox、Edge）
- 响应式布局，适配不同屏幕尺寸

### 3.3 性能
- 页面加载时间 < 3秒
- 支持并发用户数 >= 100

## 4. 运行环境

### 4.1 开发环境
- 操作系统：Windows 10/11
- JDK：17+
- Maven：3.8+
- MySQL：8.0+
- Node.js：18+
- IDE：IntelliJ IDEA / VS Code

### 4.2 技术栈
- 后端：Spring Boot 3.2 + MyBatis-Plus 3.5
- 前端：Vue 3 + Element Plus + Vite
- 数据库：MySQL 8.0
- 认证：JWT (jjwt 0.12)

# 云服务器部署指南

## 📋 部署前检查清单

| 检查项 | 状态 | 说明 |
|--------|------|------|
| ✅ JDK 17+ | `java -version` | 必须为 17.x 或更高版本 |
| ✅ MySQL 8.0+ | `mysql --version` | 推荐 MySQL 8.0.33+ |
| ✅ Maven 构建完成 | `target/contact-system-1.0.0.jar` | 确保JAR文件存在 |
| ✅ 数据库初始化脚本 | `sql/init.sql` | 包含数据库创建和初始数据 |
| ✅ SSH 访问权限 | `ssh root@10.211.55.20` | 确保能SSH登录服务器 |

## 🚀 部署步骤

### 步骤 1: 上传文件到服务器

使用 WinSCP 或命令行上传以下文件：

```
E:\py\1\contact-system\backend\target\contact-system-1.0.0.jar
E:\py\1\contact-system\sql\init.sql
E:\py\1\contact-system\deploy\deploy-to-cloud.sh
```

**WinSCP 上传方法：**
1. 打开 WinSCP
2. 输入服务器信息：
   - 主机名：`10.211.55.20`
   - 用户名：`root`
   - 密码：`your_password`
3. 将文件拖拽到 `/home/root/contact-system/` 目录

### 步骤 2: 配置服务器环境

```bash
# 登录服务器
ssh root@10.211.55.20

# 创建项目目录
mkdir -p /home/root/contact-system

# 进入项目目录
cd /home/root/contact-system

# 设置执行权限
chmod +x deploy-to-cloud.sh

# 检查文件权限
ls -la
# 应显示 deploy-to-cloud.sh 有 x 权限
```

### 步骤 3: 修改部署脚本配置

编辑 `deploy-to-cloud.sh` 文件，修改以下配置：

```bash
# 修改 MySQL 密码（第24行）
MYSQL_PASSWORD="your_actual_mysql_password"

# 可选：修改端口（第29行）
APP_PORT=8080

# 可选：修改数据库名（第26行）
DB_NAME="contact_system"
```

### 步骤 4: 执行部署脚本

```bash
# 赋予执行权限
chmod +x deploy-to-cloud.sh

# 执行部署
./deploy-to-cloud.sh

# 查看实时日志
tail -f contact-system.log
```

### 步骤 5: 验证部署结果

```bash
# 检查进程是否运行
ps -ef | grep contact-system

# 检查端口监听
netstat -tuln | grep :8080

# 检查日志
tail -20 contact-system.log
```

## 🔍 常见问题排查

### ❌ 问题：MySQL连接失败
**原因：** MySQL密码错误或用户权限不足
**解决方案：**
```bash
# 登录MySQL
mysql -u root -p

# 授权用户
GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost' IDENTIFIED BY 'your_password';
FLUSH PRIVILEGES;
```

### ❌ 问题：应用启动后无法访问
**原因：** 防火墙阻止了8080端口
**解决方案：**
```bash
# CentOS 7 开放端口
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --reload

# 或临时关闭防火墙（不推荐）
systemctl stop firewalld
```

### ❌ 问题：数据库初始化失败
**原因：** init.sql 文件路径错误
**解决方案：**
```bash
# 确保在正确目录执行
cd /home/root/contact-system
# 确保init.sql在同一目录
ls -la sql/init.sql
```
